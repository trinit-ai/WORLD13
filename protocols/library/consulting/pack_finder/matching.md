# Pack Matching

## Purpose
Score available packs against the user's stated needs and present recommendations.

## Matching Process

1. Read [REGISTRY CONTEXT] for the full catalog
2. Score each visible pack against the user's stated:
   - Industry alignment
   - Use case match
   - Feature requirements
   - Verb type fit (intake, review, scenario, compose, debrief, train)
3. Rank by relevance
4. Present top 1-3 matches

## Presentation Format

For each recommended pack:
- **Name and tagline**: What it does in one line
- **Why it fits**: Specific connection to their stated needs
- **Cartridge overview**: Brief description of the conversation flow
- **Category and verb type**: Classification context

Example:
"**Lead Qualification** — BANT/MEDDPICC lead scoring across 5 cartridges. This fits because you mentioned needing to qualify inbound leads before routing to sales. It captures contact info, scores the opportunity, and generates a qualification report."

## When No Match Exists

If no pack closely matches the user's needs:
1. Acknowledge the gap honestly
2. Suggest the closest alternatives with caveats
3. Mention Pack Builder as an option for custom packs
4. Offer to connect them with the team

## Transition

After presenting matches, ask which one interests them most: "Would you like to see how any of these work in practice?"
