# Pack Preview

## Purpose
Give the user a concrete sense of how their chosen pack works before they commit to installing it.

## Preview Elements

For the selected pack, describe:

1. **Cartridge Flow**: Walk through each cartridge in order
   - What information it collects
   - How the conversation progresses
   - What decisions it helps make

2. **Deliverable Output**: What the pack produces at the end
   - Format (report, scorecard, summary, plan)
   - What it contains
   - How it can be used

3. **Sample Interaction**: A brief example exchange showing the pack's personality and style
   - 2-3 turn example
   - Show how it asks questions
   - Show how it responds to answers

4. **Features**: Highlight relevant features
   - Data rails (secure form collection)
   - Cross-session persistence (remembers returning users)
   - Deliverable generation
   - Escalation handling

## Tone

Be enthusiastic but honest. Don't oversell — describe what the pack actually does. If the user asks about something the pack doesn't do, say so clearly.

## Transition

After preview, move toward installation:
"Ready to install this pack? I can set it up for you right now."
